# Next.js 全栈 + Supabase，AI 辅助开发

技术栈选择 Next.js 全栈框架，后端使用 Supabase（数据库、认证、实时同步），画布交互使用 React Flow。由开发者借助 AI 工具独立搭建。

**选择的理由：**

产品定位为垂直领域的 SaaS 小产品，初期由一人借助 AI 工具开发。Next.js + Supabase 的组合将前后端统一在一个框架内，Supabase 开箱即用地提供了数据库、用户认证和实时同步能力，避免了从零搭建后端的工程开销。React Flow 是专门为节点连线图设计的开源库，与信号路由规划的画布交互高度匹配。

**否决的方案：**

传统 React + 独立后端（如 Express + PostgreSQL）：工程复杂度更高，对一人团队来说维护负担重。低代码平台（如 Retool）：画布拖拽连线交互难以实现，且受平台能力上限约束，后期扩展性不足。

**约束条件：**

开发者非专职程序员，借助 AI 工具开发，因此技术栈必须对 AI 辅助编码友好——框架主流、文档丰富、社区活跃。Next.js 和 Supabase 均满足此条件。
