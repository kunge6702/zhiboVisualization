# 拓扑图可带离是 v1 必须项

接线方案的拓扑图那一半必须能在 v1 产出可带离屏幕的形式（导出图片或可打印页），否则接线方案这个术语的两半定义只兑现了一半。

**选择的理由：**

CONTEXT.md 定义"接线方案"由连线清单和拓扑图两部分组成，并明确拓扑图用途是"现场排查"。当前代码只实现了连线清单的文本导出（复制到剪贴板），拓扑图的可视化只存在于编辑器画布上，无法带离屏幕。

这造成现场排查场景的产物断裂：现场技术人员拿到 JSON 归档（ADR-0005）没用，逐根对照连线清单排查故障效率低——他要的是一张能一眼看清"这条线在整体里什么位置"的全局图。清单管执行，拓扑图管排查，职责不同，缺了拓扑图，排查这一半场景没有产物可用。

ADR-0005 解决了数据迁移，本 ADR 解决可视化产物带离，连线清单解决执行对照——三者合起来才是"从办公室到现场"的完整产物链。拓扑图带离是这个链上不可缺的一环。

技术实现（svg 转 png、打印样式、分页策略、文件命名）属于实现选型，不在本 ADR 范围内。本 ADR 只锁定产品语义：v1 必须交付可带离的拓扑图，具体技术方案由实现层决定。

**否决的方案：**

把拓扑图降级为"屏幕辅助视图"，从接线方案定义里摘掉带离要求。等于用删文档承诺来掩盖产品缺口——CONTEXT.md 白纸黑字写了拓扑图用于现场排查，删掉它就是让文档迁就现状的缺口，而非让现状补齐文档的承诺。grill 的原则是让文档反映该做的事，而非把该做的事从文档里删掉。

**约束条件：**

- 拓扑图带离形式不限（图片或可打印页均可），但必须能在没有本工具的设备上查看——现场可能只有手机或打印纸。
- 拓扑图与连线清单的编号必须一致，便于现场在图与清单之间来回对照。
- 拓扑图导出不得依赖网络，与 ADR-0001 的 PWA 离线要求兼容。
