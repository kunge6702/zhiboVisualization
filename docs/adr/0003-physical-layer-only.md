# 只管物理层，不管逻辑层

第一版只管理设备之间的物理接线（HDMI、USB、音频线等），不管理 vMix 等软件内部的通道配置、图文叠加、推流设置等逻辑层内容。

**选择的理由：**

用户的核心痛点是"到现场接线别翻车"——物理层的规划错误会导致现场拔线重接，直接暴露不专业。逻辑层（vMix 内部配置）的试错成本极低，坐在笔记本前点几下就能改，不影响现场观感。逻辑层的建模复杂度是另一个量级的，对第一版来说投入产出比不合理。

**否决的方案：**

同时管理物理层和逻辑层。需要把 vMix 的界面逻辑、通道配置、推流参数等全部搬进工具，开发周期过长，且逻辑层的配置因项目而异，难以标准化。
