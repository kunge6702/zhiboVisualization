# 直播信号路由规划工具

为直播服务团队提供的可视化信号路由规划与校验工具。在到达现场之前，通过拖拽设备、连线的方式设计信号链路，自动校验端口数量与路由完整性，输出可执行的接线方案。

## Scope

工具只管理物理层——设备之间的实际接线（HDMI、USB、音频线等）。不管理逻辑层——vMix 等软件内部的通道配置、图文叠加、推流设置等。

HDMI 等接口物理上可同时传输视频与嵌入音频，但本工具只按单一信号类型建模端口与连线——音频走独立物理链，HDMI 中的嵌入音频不在建模范围内。

系统中的信号链路分为三类：
- 现场链路：切换台输出 → 大屏，由切换台操作员控制
- 直播链路：切换台输出 → 采集卡 → 笔记本(vMix) → 直播平台，由 vMix 操作员控制
- 返送链路：笔记本(vMix) → 切换台输入，用于将远程连线画面送回现场

## Language

**信号路由 (Signal Route)**:
一条从起始设备到目标设备的完整链路，经过若干设备端口。例如：摄像机 → 切换台输入1 → 切换台输出1 → 采集卡 → 笔记本。
_Avoid_: 信号流、链路、通路

**设备 (Device)**:
信号链路中的一个节点，具有固定数量的输入/输出端口。例如：切换台、采集卡、调音台、声卡。
_Avoid_: 器材、硬件、节点

**设备库 (Device Library)**:
预设的常用设备集合，每个设备已配好端口数量与类型。支持自定义扩展新设备。团队可随项目积累复用。
_Avoid_: 设备列表、器材库、模板库

**端口 (Port)**:
设备上的物理接口，分为输入端口和输出端口，承载单一类型的信号（视频或音频）。
_Avoid_: 接口、通道、口子

**连线 (Connection)**:
两个端口之间的一条有向连接，从输出端口指向输入端口。连线精确到端口级别，与现场实际接线一一对应。
_Avoid_: 线缆、线路、跳线

**接线方案 (Wiring Plan)**:
规划工具的输出产物，由两部分组成：连线清单（逐根描述每根线的起止端口，供现场逐根对照接线）和拓扑图（设备与连线的全局可视化，供办公室审视和现场排查）。以连线清单为主要执行依据。
_Avoid_: 接线图、连线表、拓扑图

**端口校验 (Port Validation)**:
检查单条连线是否正确占用了设备上的可用端口，端口耗尽时阻止继续连线并提示。只管"画线时这个端口有没有被别的连线占"，不管多条信号需求加起来的端口容量是否充足。
_Avoid_: 端口检查、容量检查

**端口冲突 (Port Conflict)**:
两条信号需求路由竞争同一物理端口，导致其中一条无法满足。冲突不是断链——拓扑上路径存在，但端口容量不够。解法是换端口、换路由或加设备，而非加连线。
_Avoid_: 端口争用、端口碰撞

**路由完整性校验 (Route Completeness Validation)**:
检查用户声明的每一条信号需求是否存在一条从源到终点的完整路由，且不与其他需求争抢物理端口。校验结果三态：满足（找到无冲突路由）、断链（拓扑上根本没路，需加连线）、冲突（有路但端口被占，需换端口/加设备）。
_Avoid_: 通路检查、链路检查

**信号需求 (Signal Requirement)**:
用户从甲方获取的一条信号传输要求，严格定义一个起始设备、一个目标设备、以及信号类型（视频/音频）。起始设备可以是任何有输出端口的设备，目标设备可以是任何有输入端口的设备——设备在需求里扮演起始还是目标，由信号流向决定，不由设备类目决定。每条需求只对应一条起始到目标的路径，校验结果为满足、断链或冲突三态之一。同一起始到多个目标须拆分为多条需求。注意"信号源"只是设备库的一个类目名（摄像机、麦克风），与需求里的"起始设备"是不同维度的概念。
_Avoid_: 需求条目、任务项、连线需求

**项目 (Project)**:
工具中的顶层组织单元，对应一次具体的直播服务任务。每个项目包含独立的设备清单、需求清单和布线方案。支持复制已有项目来快速创建新项目。
_Avoid_: 方案、任务、工程

## Devices

初始设备库包含以下设备：

**摄像机（索尼摄像机、索尼A7M3、佳能6D2）**:
视频源设备。各有 1 个 HDMI 输出端口。

**麦克风**:
音频源设备。1 个 XLR 输出端口，接入调音台或声卡。覆盖手持、领夹、电容等接调音台的麦克风，不区分拾音头类型；USB 麦克风和无线接收器暂不在设备库中。

**切换台**:
硬件视频切换设备。4 个输入端口、2 个输出端口、1 个监看输出端口。可将任意输入路由到任意输出。

**采集卡**:
将 HDMI 视频信号转换为 USB 信号供笔记本使用。1 个 HDMI 输入端口、1 个 USB 输出端口、1 个 HDMI 环出端口。

**笔记本**:
运行 vMix 软导播的主机。1 个 HDMI 输出端口（vMix 输出画面）、多个 USB 端口（连接采集卡和声卡）。既是需求的目标设备（通过 USB 接收采集卡和声卡的信号），也是需求的起始设备（通过 HDMI 输出返送画面）——设备在需求里扮演起始还是目标，由信号流向决定。

**声卡**:
USB 音频接口。1 个音频输入端口、1 个 USB 输出端口（连接笔记本）。

**调音台**:
音频混合设备。2 个输入端口（接麦克风/声卡等音频源）、1 个监听总输出端口。

**大屏**:
现场显示设备。1 个输入端口。
